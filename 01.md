# ‚úÖ Difference between `String`, `StringBuffer`, and `StringBuilder`

---

## üîπ 1. `String`

### üìå Definition

* `String` is **immutable**
* Once created, its value **cannot be changed**

---

### üìå Example

```java
String s = "Java";
s = s.concat(" World");
```

### üîç What happens internally?

* `"Java"` remains unchanged
* A **new object** `"Java World"` is created
* Old object becomes eligible for GC

---

### üìå Characteristics

| Feature     | String                          |
| ----------- | ------------------------------- |
| Mutability  | ‚ùå Immutable                     |
| Thread-safe | ‚úÖ Yes                           |
| Performance | ‚ùå Slow when modified frequently |
| Memory      | Uses String Constant Pool       |

---

### üìå When to use `String`?

‚úî When data **does not change frequently**
‚úî When **security & thread-safety** are important
‚úî Example: usernames, passwords, URLs

---

### üìå Interview Line

> `String` is immutable, which makes it thread-safe and secure.

---

## üîπ 2. `StringBuffer`

### üìå Definition

* `StringBuffer` is **mutable**
* **Thread-safe** (synchronized)

---

### üìå Example

```java
StringBuffer sb = new StringBuffer("Java");
sb.append(" World");
```

‚û°Ô∏è Same object is modified

---

### üìå Characteristics

| Feature         | StringBuffer                |
| --------------- | --------------------------- |
| Mutability      | ‚úÖ Mutable                   |
| Thread-safe     | ‚úÖ Yes                       |
| Performance     | ‚ùå Slower than StringBuilder |
| Synchronization | ‚úÖ Yes                       |

---

### üìå When to use `StringBuffer`?

‚úî In **multi-threaded environments**
‚úî When multiple threads modify the same string
‚úî Example: Banking applications, financial systems

---

### üìå Interview Line

> `StringBuffer` is thread-safe due to synchronization.

---

## üîπ 3. `StringBuilder`

### üìå Definition

* `StringBuilder` is **mutable**
* **Not thread-safe**
* Introduced in **Java 1.5**

---

### üìå Example

```java
StringBuilder sb = new StringBuilder("Java");
sb.append(" World");
```

‚û°Ô∏è Fastest option for string manipulation

---

### üìå Characteristics

| Feature         | StringBuilder |
| --------------- | ------------- |
| Mutability      | ‚úÖ Mutable     |
| Thread-safe     | ‚ùå No          |
| Performance     | ‚úÖ Fastest     |
| Synchronization | ‚ùå No          |

---

### üìå When to use `StringBuilder`?

‚úî In **single-threaded** environments
‚úî When **performance is critical**
‚úî Example: Logging, report generation, loops

---

### üìå Interview Line

> `StringBuilder` is faster because it is not synchronized.

---

## üî• Internal Comparison (Memory & Performance)

### üîç Performance Order

```
StringBuilder > StringBuffer > String
```

---

### üîç Memory Behavior

```java
String s = "A";
s = s + "B";
s = s + "C";
```

‚û°Ô∏è Creates **multiple objects**

```java
StringBuilder sb = new StringBuilder("A");
sb.append("B").append("C");
```

‚û°Ô∏è Only **one object**

---

## üìä Final Comparison Table

| Feature          | String   | StringBuffer | StringBuilder |
| ---------------- | -------- | ------------ | ------------- |
| Mutable          | ‚ùå        | ‚úÖ            | ‚úÖ             |
| Thread-safe      | ‚úÖ        | ‚úÖ            | ‚ùå             |
| Performance      | ‚ùå        | ‚ö†Ô∏è           | ‚úÖ             |
| Synchronization  | ‚ùå        | ‚úÖ            | ‚ùå             |
| Memory Efficient | ‚ùå        | ‚úÖ            | ‚úÖ             |
| Introduced in    | Java 1.0 | Java 1.0     | Java 1.5      |

### üè¶ Banking

## üí° Interview One-Line Summary
